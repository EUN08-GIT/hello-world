Oracle은 SQL 문장을 처리할 때 다음과 같은 흐름으로 캐시를 활용합니다:

SQL 문장 해시 생성

SQL 텍스트를 기반으로 해시 키 생성

이 키를 사용해 Library Cache에서 기존 실행 계획이 있는지 확인

캐시 조회

동일한 SQL 문장이 이미 실행된 적 있다면, 해당 실행 계획을 재사용

이때 바인드 변수가 동일해야 커서 재사용 가능

없으면 파싱 후 저장

실행 계획이 없으면 새로 파싱하고, Library Cache에 저장

커서 관리

세션 단위로 Cursor Cache에 커서 정보 저장

커서가 닫히면 Library Cache에서 제거될 수 있음


커서가 뭘 말하는건지 뭘 가르킨다는 뜻에서 가져온 단어인거야?

ㅇㅇ 이 흐름을 실제 V$SQL, V$SQL_PLAN, V$RESULT_CACHE_OBJECTS 뷰로 확인하는 방법을 알려줘

뭐 소프트파싱 하드파싱 문장파싱 이런것들을 너가 말해줬는데 파싱이라는게 머야?

1. SQL 문장 파싱
사용자가 SQL을 실행하면 Oracle은 해당 문장을 Library Cache에서 검색

없으면 하드 파싱을 통해 실행 계획 생성 → Library Cache에 저장
이거 할때 해싱으로 키값에는 문장넣고 밸류값에는 실행계획을 넣는거야 아니면 그냥 리스트 처럼 넣는거야?
